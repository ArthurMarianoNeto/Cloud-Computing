Forte™ for Java™ Release 2.0
para instalação em Linux

Leia a documentação a seguir antes de você instalar o Forte™ for Java™ Release 2.0 a partir do CD.

Você leu o contrato de licenciamento do Forte™ for Java™ Release 2.0?

Você deve ter Java™ 2 SDK, Standard Edition, v. 1.3 instalado em seu sistema para poder instalar e executar o Forte for Java.

Documentação de Java:

A documentação do Java 2 SDK pode ser baixada como arquivo zip para Linux do endereço:
http://java.sun.com/j2se/1.3/docs.html
As instruções de instalação para a documentação podem ser encontradas no endereço:
http://java.sun.com/j2se/1.3/install-docs.html

Se você quiser ter acesso à documentação de Java 2 SDK a partir do Forte for Java, baixe, instale e monte esta documentação como um sistema de arquivos no IDE.

Para montar a documentação de Java como um sistema de arquivos dentro do Forte for Java (depois de instalar o Forte for Java),:

  1. Abra a janela do Forte for Java Explorer: View > Explorer.

  2. Clique na aba Javadoc na base da janela do Explorer.

  3. Clique com o botão direito no nodo Javadoc dentro da janela do Explorer.

  4. Selecione Mount Directory do menu pop-up.

  5. Navegue até o diretório no qual você instalou a documentação.
          C:/jdk1.3/docs
    e selecione-o com um clique simples.

  6. Clique em Mount.

A documentação agora estará acessível de dentro do IDE através da aba Javadoc na janela do Explorer.

Instalação como usuário único para o Linux Red Hat 6.2 e outras plataformas Linux

Os arquivos para instalar o Forte™ for Java™ estão localizados no diretório software-lin\forte no nível do diretório-raiz deste CD-ROM.

1. Baixe o arquivo do CD para um diretório em seu computador pessoal:

forte_ce_2.sh

2. Mude o seu diretório de trabalho para este diretório.

3. Inicie a instalação (como um usuário não-"root") executando este arquivo:

$ sh ./forte_ce_2.sh

4. O programa de instalação vai verificar se há uma Máquina Virtual Java compatível instalada.

depois, clique em Next

Nota: se o programa de instalação não encontrar uma MVJ compatível, aparecerá um menu solicitando que você escolha uma das duas opções seguintes:

Selecione 0 para especificar um caminho para a sua MVJ.
Selecione 1 para terminar a instalação.

5. A mensagem: Extracting Installation class aparece por um momento antes de a tela Welcome ser exibida.

Selecione Next a partir da tela Welcome para continuar a instalação.

6. Aparece a tela Readme.

Leia este documento, depois selecione Next.

7. Aparece o contrato de licenciamento. Após revisar o contrato,

selecione o botão de rádio Agree
clique em Next para continuar instalando o Forte for Java.

8. Uma lista de Máquinas Virtuais Java instaladas é exibida.

Selecione JDK 1.3 e clique em Next.

9. Selecione o diretório de instalação default.

10. Clique em Install Now.

11. Após os arquivos serem instalados, aparece a janela Installation Summary.

Após revisar o resumo, clique em Exit.

12A. Inicie o Forte for Java:

$ /forte4j/bin/runide.sh
Nota: o arquivo runide.exe ignora o valor da variável de ambiente CLASSPATH. Isto afeta o acesso a bancos de dados a partir do Forte for Java. Caso execute o Forte for Java sem a variável CLASSPATH, você terá que adicionar manualmente os arquivos de "drivers" de bancos de dados a

<FFJ_HOME>/libext.

12B. Você pode iniciar o Forte for Java a partir da linha de comando com a variável CLASSPATH (usando a opção -cp):

$ runide.sh -cp${CLASSPATH}

Desinstalando o Forte for Java

1. Execute o programa de instalação.

2. Abra a janela de File Deletion Confirmation.

3. Clique em OK para desinstalar o Forte for Java.

Instalação como usuário único através de um arquivo .class de Java

1. Salve o arquivo .class de Java a partir do CD em um diretório em seu computador pessoal.

Ele está localizado no diretório software-lin\forte no nível do diretório-raiz deste CD-ROM.

2. Mude seu diretório de trabalho para o diretório no qual o arquivo .class de Java está localizado.

Nota: se você tiver uma CLASSPATH configurada e ela não incluir o diretório atual, adicione o diretório atual à sua CLASSPATH existente.

Se você não tiver uma CLASSPATH configurada, pule a próxima etapa.

(Para verificar se você tem uma CLASSPATH configurada atualmente, digite set em um prompt de comando. Se você ver a CLASSPATH na lista, você tem uma CLASSPATH configurada.)

3A. Para adicionar o diretório atual à sua CLASSPATH em um shell Bourne (que normalmente dá um prompt $), digite:

$ CLASSPATH=.:$CLASSPATH; export CLASSPATH

3B. Para adicionar o diretório atual à sua CLASSPATH em um shell C (que normalmente dá um prompt % ou >), digite:

% setenv CLASSPATH .:$CLASSPATH

4. Digite o seguinte comando (sem a extensão .class):

java forte_ce_2

Nota: isto parte do princípio de que você tem o JDK 1.3 em seu caminho. Se não tiver, você deve especificar o caminho completo para o executável do interpretador Java:

<full_path_to_java_interpreter> forte_ce_2

Isto inicia a rotina de instalação.

5. Trabalhe com as caixas de diálogo para completar a instalação.

6. Após completada a instalação, use o script runide em seu diretório de instalação para iniciar o Forte for Java.

7A. Você pode iniciar o Forte for Java a partir da linha de comando:

$ runide.sh

Nota: o runide.exe ignora o valor da variável de ambiente CLASSPATH. Isto afeta o acesso a bancos de dados a partir do Forte for Java. Se você executar o Forte for Java sem a variável CLASSPATH, você terá que adicionar manualmente os arquivos de "drivers" de bancos de dados a:

<FFJ_HOME>/libext.

7B. Você pode iniciar o Forte for Java a partir da linha de comando com a variável CLASSPATH (usando a opção -cp):

$ runide.sh -cp${CLASSPATH}

Desinstalando o Forte for Java precedente

1. Execute o uninstall.class localizado em /forte4j.

2. Siga as instruções das caixas de diálogo para completar a desinstalação.

Instalação como um root do ForteCE-FCS Forte for Java

1. Configure a variável JAVA_PATH para o diretório "home" de Java.

2. Faça "login" como superusuário

$ su
#

3. Instale os arquivos RPM (como superusuário)

# rpm -i ForteCE-FCS-2[1].noarch.rpm

4.  Faça logout como superusuário

# exit
$

5. Inicie o Forte for Java

$ ./runide

Montando um arquivo JAR ou ZIP como um sistema de arquivos Javadoc no IDE

Para montar um arquivo JAR ou ZIP (sem executar comandos unzip ou jar) como um sistema de arquivos Javadoc:

1. Abra a janela Explorer no IDE.

2. Clique na aba Javadoc na base da janela do Explorer.

3. Dê um clique com o botão direito no nodo Javadoc e selecione Mount JAR.

4. Navegue até o arquivo JAR ou ZIP e selecione-o com um clique simples.

5. Clique em Mount.

     Se o IDE não encontrar os arquivos de índice de Javadoc, o sistema lhe perguntará se você quer continuar.

6.  Clique em Yes para continuar.

A documentação agora estará disponível através da aba Javadoc.

Removendo um projeto através do sistema operacional

1.  No diretório de instalação do IDE, navegue até o system/project e remova o diretório que contém o projeto que você quer remover.

2.  Remova o arquivo project.last do diretório system.

ATENÇÃO: se você usar este método enquanto o IDE estiver sendo executado, você corre o risco de remover o projeto atualmente aberto.

Removendo um projeto através do IDE

1. Escolha Project | Settings a partir da janela principal.

2. Na janela Project | Settings,

expanda o nodo Filesystems Settings.

Escolha o nodo Default System.

3. Configure a propriedade Hidden para False.

O sistema de arquivos Default System será exibido no Explorer.

4.  No Explorer:

expanda o nodo Default System

expanda o nodo Projects

selecione o projeto que você quer remover.

5. Em Default System / Projects

clique com o botão direito no projeto que você quer remover.

6.  Escolha Delete a partir do menu contextual.

O projeto default não pode ser removido.

7.  Na janela Project | Settings

configure a propriedade Hidden para o sistema de arquivos default do sistema novamente para True.

ATENÇÃO: o sistema de arquivos default do sistema consiste em arquivos que o próprio IDE usa. Você deve manter este sistema de arquivos oculto para que você não altere ou remova acidentalmente estes arquivos.

Subdiretórios de instalação

/beans contém os componentes JavaBeans™ instalados no Forte for Java.
/bin inclui o iniciador do Forte for Java, o arquivo ide.cfg e, em instalações no Windows, o addtopath.bat, que é usado pelo arquivo de inicialização em batch.
/Development o diretório montado por default em Filesystems. Os objetos que você cria em Forte for Java serão salvos aqui, a menos que você monte outros diretórios e os utilize.
/docs  contém a documentação do Forte for Java.
/javadoc o diretório montado por default no repositório javadoc de Forte for Java. A documentação javadoc que você cria em Forte for Java será armazenada aqui.
/lib  contém arquivos JAR que compreendem a implementação do núcleo do IDE e as APIs abertas.
/lib/ext contém extensões do IDE para outras funcionalidades, como
  • JavaHelp
  • leiaute absoluto
  • javac
  • expressões regulares.
/lib/patch quaisquer arquivos JAR ou ZIP incluídos nesta pasta serão automaticamente incluídos no início do caminho para as classes de inicialização do IDE; será uma alteração do núcleo.
/modules qualquer arquivo JAR nesta pasta é um módulo do IDE.
/modules/ext contém bibliotecas usadas pelos módulos.
/sources contém fontes para as bibliotecas que podem ser redistribuídos com aplicativos do usuário.
/system inclui arquivos e diretórios usados pelo IDE para finalidades especiais. Ele inclui
  • ide.log, fornece informações úteis ao se solicitar suporte técnico,
  • project.basic, e
  • project.last, os quais contêm informações sobre projetos do Forte para Java.

É montado como um sistema de arquivos oculto.
/system/actions contém ações que aparecem em Global Options, em Actions.
/system/applet contém arquivo de regras para depuração de applets.
/system/parserDB armazena bancos de dados que são usados para completar código Java e outras funções do Editor.
/system/startup armazena classes que são executadas quando o IDE é iniciado.
/temp
