Java™ 2 SDK,
Standard Edition v 1.3.1
para instalação em Linux

Leia a documentação a seguir antes de você instalar o Java™ 2 SDK a partir do CD.

Você leu o contrato de licenciamento do Java™ 2 SDK?

Requisitos de sistema

Esta versão do Java 2 SDK é suportada em plataformas Intel Pentium que rodam o kernel do Linux v 2.2.12 e glibc v2.1.2-11 ou posterior. É necessário um mínimo de 32 megabytes de RAM. Recomenda-se 48 megabytes de RAM, modo de cores de 16 bits, gerenciadores de janelas KDE e KWM usados em conjunto com os monitores configurados para hosts locais.

Verifique sua versão de glibc com o seguinte comando:

ls /lib/libc-*

Você deve ter cerca de 75 megabytes de espaço livre em disco antes de tentar intalar o software do Java 2 SDK. Caso também instale o pacote separado de documentação que foi baixado, você precisará de 125 megabytes adicionais de espaço livre em disco.

A plataforma Linux suportada oficialmente para o J2SDK 1.3.1 é a RedHat Linux 6.2 e a maior parte dos testes do J2SDK 1.3.1 para Linux foram feitos naquele sistema operacional. Entretanto, o J2SDK 1.3.1 passou por testes limitados em outros sistemas operacionais Linux. A tabela a seguir mostra as plataformas Linux, por locale, nas quais a Sun testou esta versão pelo menos de maneira limitada. Veja abaixo as informações sobre os problemas conhecidos em vários sistemas operacionais Linux.

Problemas conhecidos

O RedHat Linux 6.2 é a plataforma Linux suportada oficialmente pelo J2SDK 1.3.1. Testes limitados foram feitos em outros sistemas operacionais Linux, e os que seguem são problemas conhecidos nas plataformas não-suportadas.

  • Se você usa o Red Hat Linux 7, recomendamos a versão 7.1 em vez da 7.0. Testes limitados revelaram problemas quando se usa o J2SDK com o Red Hat Linux 7.0. Alguns deles estão descritos abaixo.


  • Quando System.exit(int) é invocado no Red Hat 7.0, o programa nunca termina com um valor diferente de zero. Este problema ocorre aparentemente devido a um erro na função exit na biblioteca libc.so. Para evitar este problema, use a versão suportada do Red Hat, versão 6.2, ou, se você quer usar o Red Hat 7, use a versão 7.1 em vez da 7.0.


  • No RedHat Linux 7.0, se você quiser usar o Classic VM em vez de um dos Java HotSpot VMs no J2SDK 1.3.1, você deve baixar e instalar o arquivo glibc-2.2-9.i386.rpm disponível no endereço http://www.redhat.com/support/errata/RHBA-2000-079.html. Não instale os arquivos i686 disponíveis na mesma página da Web, pois eles impedirão o funcionamento adequado dos Java HotSpot VMs no J2SDK 1.3.1.


  • Se usa o RedHat Linux 7 Server, você deve instalar manualmente compat-libstdc++-6.21-2.9.0.0.i386.rpm para evitar o "error while loading shared libraries" quando estiver usando os Java HotSpot VMs. Este arquivo está localizado no diretório /RedHat/RPMS no CD-ROM do RedHat Linux 7. Você também pode obter uma cópia deste arquivo do endereço http://rpmfind.net. Para instalar o arquivo, use este comando:
    rpm --install compat-libstdc++-6.21-2.9.0.0.i386.rpm
    Não é necessário instalar manualmente compat-libstdc++-6.21-2.9.0.0.i386.rpm se você estiver usando o RedHat Linux 7 Workstation.


  • Ao usar versões do RedHat Linux diferentes de 6.1, o arquivo font.properties pode deixar de exibir alguns caracteres Symbol/Dingbats apropriadamente em alguns componentes AWT. Para corrigir este problema, use este arquivo font.properties revisado para substituir aquele em <JAVA_HOME>/jre/lib/.


  • O Caldera OpenLinux usa a versão 2.1.2-3 de glibc. Como aquela versão não é mais recente ou igual a 2.1.2-11, o instalador rpm do Java 2 SDK vai acusar um erro durante a verificação de dependências. Recomendamos que você obtenha uma versão atualizada da biblioteca glibc disponível a partir de Caldera nos seguintes locais:
    • ftp.caldera.com:/pub/updates/eDesktop/2.4/current/RPMS

Instruções de instalação

O Java™ 2 SDK, Standard Edition, v1.3.1 está disponível em dois formatos de instalação. Os dois arquivos estão localizados no diretório software-lin\sj-sdk no nível do diretório-raiz deste CD-ROM.

  • Um arquivo binário de auto-extração denominado j2sdk-1_3_1-linux-i386.bin que pode ser usado para instalar o Java 2 SDK em qualquer lugar que você escolher. Se você estiver usando este arquivo, veja Instalação do código binário de auto-extração abaixo.


  • O arquivo j2sdk-1_3_1-linux-i386-rpm.bin, que contém pacotes RPM que compreendem o Java 2 SDK. Se você estiver usando este pacote, veja Instalação do arquivo RPM abaixo.

Os pacotes para os dois formatos de instalação estão empacotados no script de shell .bin que exibe os contratos de licenciamento dos produtos antes da instalação propriamente dita.

O Java 2 SDK inclui o Java Plug-in. Veja Instalação do Java Plug-in abaixo.

Instalação do código binário de auto-extração

Use estas instruções se você quiser usar o arquivo binário de auto-extração para instalar o Java 2 SDK. Se desejar instalar pacotes RPM que compreendem o SDK, use Instalação do arquivo RPM abaixo.

O arquivo binário de auto-extração (j2sdk-1_3_1-linux-i386.bin) para instalar o Java™ 2 SDK está localizado no diretório software-lin\sj-sdk no nível do diretório-raiz deste CD-ROM. Copie o arquivo do CD para seu disco rígido com a ferramenta gráfica ou de linha de comando favorita, depois siga as instruções abaixo.

1. Verifique o tamanho do arquivo baixado.

Verifique o grande arquivo que você baixou para se assegurar de que ele tenha o tamanho correto:

j2sdk-1_3_1-linux-i386.bin     26924751 bytes

Se o tamanho do arquivo não conferir, significa que provavelmente o arquivo foi corrompido durante o download. Neste caso, tente baixar o arquivo novamente.

2. Copie j2sdk-1_3_1-linux-i386.bin para o diretório no qual você quer instalar o Java 2 SDK.

3. Execute j2sdk-1_3_1-linux-i386.bin

Inicie o arquivo de programa que você baixou, j2sdk-1_3_1-linux-i386.bin, usando os seguintes comandos, a partir do diretório no qual ele está localizado:
chmod a+x j2sdk-1_3_1-linux-i386.bin
./j2sdk-1_3_1-linux-i386.bin

O script irá exibir um contrato de licenciamento de código, com o qual você deve concordar antes de a instalação continuar. Depois que você concordou com o contrato de licenciamento, o script de instalação irá instalar o Java 2 SDK em um diretório chamado jdk1.3.1 no diretório atual.

Nota sobre acesso de root - Desempacotar o software cria automaticamente um diretório chamado jdk1.3.1. Observe que, se você optar por instalar o Java 2 SDK em um lugar acessível a todo o sistema, como /usr/local, você primeiro precisa se tornar root para obter as permissões necessárias. Se você não tiver acesso de root, simplesmente instale o Java 2 SDK em seu diretório home, ou em um subdiretório no qual você tenha permissão de gravar.

Nota sobre a sobreposição de arquivos - Se você desempacotar o software ou a documentação em um diretório que contém um subdiretório chamado jdk1.3.1, o novo software irá sobrescrever os arquivos de mesmo nome naquele diretório jdk1.3.1. Tenha o cuidado de renomear o diretório antigo se ele contiver arquivos que você gostaria de manter.


Instalação do arquivo RPM

Use estas instruções se quiser instalar o Java 2 SDK na forma de pacotes RPM. Se, em vez disso, você quiser usar o arquivo binário de auto-extração, veja Instalação do código binário de auto extração acima.

O arquivo binário de auto-extração (j2sdk-1_3_1-linux-i386-rpm.bin) para instalar o Java™ 2 SDK está localizado no diretório software-lin\sj-sdk no nível do diretório-raiz deste CD-ROM. Copie o arquivo do CD para seu disco rígido com a ferramenta gráfica ou de linha de comando favorita, depois siga as instruções abaixo.

1. Verifique o tamanho do arquivo baixado.

Verifique o grande arquivo que você baixou para se assegurar de que ele tenha o tamanho correto:

j2sdk-1_3_1-linux-i386-rpm.bin     25450562 bytes

Se o tamanho do arquivo não conferir, significa que provavelmente o arquivo foi corrompido durante o download. Neste caso, tente baixar o arquivo novamente.

2. Execute o j2sdk-1_3_1-linux-i386-rpm.bin e concorde com o contrato que ele exibe.

Inicie o arquivo de programa que você baixou, j2sdk-1_3_1-linux-i386-rpm.bin, usando os seguintes comandos, a partir do diretório no qual ele está localizado:
chmod a+x j2sdk-1_3_1-linux-i386-rpm.bin
./j2sdk-1_3_1-linux-i386-rpm.bin

O script vai exibir um contrato de licenciamento de código, com o qual você deve concordar antes de a instalação continuar. Depois que você concordou com o contrato de licenciamento, o script de instalação irá criar o arquivo jdk-1.3.1.i386.rpm no diretório atual.

3. Torne-se root, executando o comando su e digitando a senha do superusuário.

4. Execute o comando rpm para instalar os pacotes:

rpm -iv jdk-1.3.1.i386.rpm

Isto vai instalar os pacotes que compreendem o Java 2 SDK.

5. Saia do shell de root.

Instalação do Java Plug-in

Para instalar o Java Plug-in, siga as seguintes instruções.

1. Desinstale a versão anterior do Java Plug-in, se for o caso.

rm -fr $HOME/.netscape/java
rm $HOME/.netscape/plugins/javaplugin.so

2. Configure a variável de ambiente NPX_PLUGIN_PATH para o diretório que contém o arquivo javaplugin.so. Este arquivo está localizado dentro do Java 2 SDK, no diretório jre/plugin/i386/ns4/.

3. Para navegadores Netscape 6: crie um link simbólico no diretório de plugins do Netscape

Se (e somente se) você estiver usando o Netscape 6, você precisará criar, no diretório plugins do Netscape, um link simbólico para o caminho do arquivo libjavaplugin_oji.so. Este arquivo está localizado dentro do Java 2 SDK, no endereço jre/plugin/i386/ns600/libjavaplugin_oji.so.
cd ${NETSCAPE6}/plugins
ln -s ${JRE}/plugin/i386/ns600/libjavaplugin_oji.so .

4. Para todos os navegadores Netscape: inicie o navegador Netscape ou reinicie-o se ele já estiver sendo executado.

5. Para todos os navegadores Netscape: reinicie o navegador uma segunda vez.

É necessário reiniciar o navegador uma segunda vez devido ao problema 4358142.
