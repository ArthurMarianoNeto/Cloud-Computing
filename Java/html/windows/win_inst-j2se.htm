Requisitos de sistema

O JavaTM 2 SDK destina-se ao uso com os sistemas operacionais Windows 95, Windows 98 (1a ou 2a edição), Windows NT 4.0 com o Service Pack 5, Windows ME, Windows 2000 Professional, Windows 2000 Server ou Windows 2000 Advanced Server, sendo executados em hardware Intel. Somente a edição Professional do Windows 2000 é suportada com "locales" diferentes de "English".

É necessário um processador Pentium 166MHz ou mais rápido, com pelo menos 32 megabytes de RAM física, para executar os aplicativos baseados em gráficos. Recomendam-se 48 megabytes de RAM para os applets executados dentro de um navegador que usa o produto Java Plug-in. Executar com menos memória pode provocar "swapping" de disco, o que tem um grave efeito sobre o desempenho. Programas muito grandes podem exigir mais RAM para desempenho adequado.

Você deve ter 70 megabytes de espaço livre em disco antes de tentar instalar o software do Java 2 SDK. Se também quiser instalar o pacote de documentação que pode ser baixado da Internet, você precisará de um espaço livre adicional de 120 megabytes em disco.

Instruções de instalação

Neste procedimento, você irá executar o programa de auto-instalação para desempacotar e instalar o pacote de software Java 2 SDK.

Nota: Depois de instalar o software Java, ele pedirá que você reinicie o sistema, o que completa as modificações do registro de que você irá precisar se quiser desinstalar este produto mais tarde. Para continuar a usar estas instruções depois de reiniciar o sistema, imprima-as agora ou use a função de histórico de seu navegador da Web para retornar a esta página.

O procedimento de instalação tem as seguintes etapas:

  1. Verifique o tamanho do arquivo de download
  2. Execute o programa de instalação do Java 2 SDK
  3. Remova o(s) arquivo(s) baixado(s) (Opcional)
  4. Atualize a variável PATH
  5. Verifique a variável CLASSPATH
  6. Comece a usar as ferramentas do SDK!
  7. Desinstalando o Java 2 SDK

Se você tiver quaisquer dificuldades, veja a seção Investigando problemas no fim deste documento ou use este link para relatar um problema de instalação.

4. Atualize a variável PATH

Você pode executar o Java 2 SDK sem configurar a variável PATH, ou, como opção, pode configurá-la como uma conveniência.

Devo configurar a variável PATH?
Configure a variável PATH se você quiser executar de forma conveniente os programas do Java 2 SDK (javac.exe, java.exe, javadoc.exe, etc.) a partir de qualquer diretório sem ter que digitar o caminho completo do comando. Caso não configure a variável PATH, você precisa especificar o caminho completo para o programa cada vez que você o executa, como em:
C:> \jdk1.3.1\bin\javac MyClass.java

É útil configurar a variável PATH permanentemente de modo que ela permaneça configurada após a reinicialização do sistema.

Como eu configuro a variável PATH permanentemente ?
Para configurar o PATH permanentemente, adicione o caminho completo do diretório jdk1.3.1\bin à variável PATH. Normalmente, este caminho completo é algo parecido com C:\jdk1.3.1\bin. Configure PATH da seguinte forma, dependendo de se você está no Windows NT ou no Windows 95/98/2000/ME.

Windows NT e Windows 2000 - Para configurar a PATH permanentemente:

  1. Escolha Configurações, Painel de Controle e selecione Sistema. No Windows NT, selecione a aba Ambiente; no Windows 2000 selecione a aba Avançadas. Procure "Path" nas Variáveis de Ambiente e Variáveis do Sistema. Se você não tiver certeza de onde adicionar o caminho, adicione-o na extremidade direita de "Path" em Variáveis do Usuário. Um valor típico para PATH é:
    2. C:\jdk1.3.1\bin

    O uso de maiúsculas ou minúsculas é indiferente. Clique em "Configurar", "OK" ou "Aplicar".

    O PATH pode ser uma série de diretórios separados por ponto-e-vírgulas (;). O Microsoft Windows procura os programas nos diretórios de PATH na ordem, da esquerda para a direita. Você deve ter somente um diretório bin para um Java SDK no caminho de cada vez (aqueles que seguem o primeiro são ignorados), de modo que, se algum já estiver presente, você pode atualizá-lo para jdk1.3.1.

  2. O novo caminho surte efeito em cada nova janela de linha de comando que você abre após configurar a variável PATH.

Windows 98, Windows 95 - Para configurar PATH permanentemente, abra o arquivo AUTOEXEC.BAT e adicione ou altere o comando PATH da seguinte maneira:

  1. Inicie o editor do sistema. Escolha "Iniciar", "Executar" e digite sysedit, e depois clique em OK. O editor do sistema inicia abrindo diversas janelas. Vá para a janela que está exibindo AUTOEXEC.BAT.

  2. Procure o comando PATH (se não houver, adicione um). Se você não tiver certeza sobre onde adicionar o caminho, adicione-o à extremidade direita de PATH. Por exemplo, no comando PATH a seguir, adicionamos o diretório bin na extremidade direita:
    3. PATH C:\WINDOWS;C:\WINDOWS\COMMAND;C:\JDK1.3.1\BIN

    Letras maiúsculas ou minúsculas são indiferentes. O PATH pode ser uma série de diretórios separados por ponto-e-vírgulas (;). O Microsoft Windows procura os programas nos diretórios de PATH na ordem, da esquerda para a direita. Você deve ter somente um diretório bin para um Java SDK no caminho de cada vez (aqueles que seguem o primeiro são ignorados), de modo que, se algum já estiver presente, você pode atualizá-lo para jdk1.3.1.

  3. Para fazer o caminho surtir efeito na janela de linha de comando atual, execute o seguinte:
    4. C:> c:\autoexec.bat

    Para descobrir qual o valor atual de seu PATH, para ver se ele surtiu efeito, digite o seguinte no prompt de comando:

    C:> path

Windows ME - Para configurar PATH permanentemente:

A partir do menu Iniciar, escolha Programas, Acessórios, Ferramentas do sistema e Informações do sistema. Isto abrirá uma janela com o título "Microsoft Help and Support". A partir daqui, escolha o menu de ferramentas, depois selecione o utilitário de configuração do sistema. Clique na aba ambiente, selecione PATH e pressione o botão de editar. Agora adicione o SDK a seu caminho como descrito na etapa b acima. Depois que você adicionou a localização do SDK a seu PATH, salve as alterações e reinicie sua máquina quando for solicitado.

5. Verifique a variável CLASSPATH

A variável CLASSPATH é uma maneira de dizer aos aplicativos escritos na linguagem de programação Java (incluindo os utilitários do Java 2 SDK) onde procurar as classes do usuário (as classes que você desenvolve). (A opção de linha de comando -classpath é a maneira preferida.) Se sua máquina não tiver a variável CLASSPATH configurada, você pode ignorar o resto desta etapa. Para verificar isto, execute o comando set a partir do prompt do DOS:
C:> set

Se CLASSPATH não aparecer na lista de configurações, ela não está configurada. Se sua variável CLASSPATH já estiver configurada para algum valor, você pode querer limpar suas configurações de CLASSPATH, portanto, continue lendo.

Devo modificar a variável CLASSPATH?
O Java 2 SDK funcionará bem mesmo se CLASSPATH estiver configurada para uma versão anterior do software do SDK, desde que ele contenha o diretório atual ".". Entretanto, se sua CLASSPATH contiver classes.zip (que existia apenas no JDK 1.0.x e no JDK 1.1.x) e você não pretende continuar usando estas versões mais antigas, você pode remover aquela configuração da CLASSPATH agora. Em qualquer caso, se CLASSPATH estiver configurada, ela deve incluir o diretório atual -- isto torna possível compilar e depois executar as classes no diretório atual.

Como eu modifico a CLASSPATH?
Use o mesmo procedimento que você usou para a variável PATH na etapa anterior e:

  1. Remova a variável de ambiente CLASSPATH inteiramente.

    2. Com o Java 2 SDK, o valor default é ".", o diretório atual. Para incluir quaisquer classes do usuário, use em vez dele a opção de linha de comando -classpath com java, javac, javadoc e outras ferramentas. Esta é a abordagem recomendada, porque ela não força uma CLASSPATH para todos os aplicativos.

    -OU-

  2. Se você tiver aplicativos que exigem que a CLASSPATH seja configurada, mantenha essas classes de usuário exigidas em CLASSPATH e inclua o diretório atual ".". Se você não estiver mais usando o JDK 1.1.x, remova classes.zip.

Para obter mais informações, consulte o endereço Setting the class path (Sun).

6. Comece a usar as ferramentas do SDK!

Seu sistema de computador agora deve estar pronto para usar o Java 2 SDK. Nesta etapa, você executará alguns comandos simples para se assegurar de que está funcionando apropriadamente.

Se você estiver apenas iniciando o desenvolvimento e a execução de programas na linguagem de programação Java, consulte o endereço The Java Tutorial on-line para obter alguma orientação. Em especial, observe as trilhas do tutorial sob o título Trails Covering the Basics.

Você também pode baixar a documentação do Java 2 SDK a partir da página de download do Java 2 SDK..

7. Desinstalando o Java 2 SDK

Se você quiser desinstalar o Java 2 SDK, vá para o menu Iniciar e selecione Configurações, depois selecione Painel de Controle. No Painel de Controle, abra o utilitário Adicionar ou Remover Programas. Uma lista de softwares que você pode desinstalar será apresentada a você. Simplesmente escolha o Java 2 SDK da lista e clique no botão "Adicionar ou Remover ...".

Investigando problemas na instalação

Abaixo estão algumas dicas para contornar os problemas que às vezes são vistos durante ou após uma instalação. Para obter mais informações sobre a investigação de problemas, consulte a Java FAQ.

  • Arquivo ActiveX Control "danificado"

    • Quando você instala a versão 1.3.1 do Java Runtime Environment, o arquivo ActiveX Control em C:\Winnt\Downloaded Program Files\Java Runtime Environment 1.3.1 terá um estado "danificado". Esta é meramente uma questão aparente, e o estado "danificado" não deve ter nenhum efeito adverso em qualquer situação.

    Não obstante, existe uma maneira de contornar o problema se, por alguma razão, você não puder ter um estado de "danificado". Vá para "Iniciar -> Executar" e digite "regedit". A partir das janelas do regedit, navegue para a chave do registro "HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Code Store Database\Distribution Units\8AD9C840-044E-11D1-B3E9-00805F499D93\DownloadInformation". Dê um duplo clique no string INF e remova o string salientado sob "Dados do valor". O arquivo ActiveX Control terá agora um estado de "instalado".

  • O programa de instalação diz que você não tem espaço em disco suficiente, embora você tenha mais de 4GB de espaço disponível no disco

    • Este problema pode ocorrer se o espaço livre em seu disco estiver no intervalo estreito [4GB, 4GB + espaço necessário -1]. Para contornar este problema, crie um arquivo (ou arquivos) temporário(s) que ocupe(m) espaço em disco suficiente para tornar o espaço disponível menor do que 4GB, depois prossiga com a instalação. Depois que a instalação for completada, você pode remover o(s) arquivo(s) temporário(s).

    Um problema similar pode ocorrer com quantidades maiores de espaço livre em disco. Consulte o relatório de problemas 4406594.

  • Se você ver a seguinte mensagem de erro no Windows 2000
    • config.nt. The system file is not suitable for running MS-DOS and Microsoft Windows Applications.

    ela indica um problema com o arquivo %SystemRoot%\System32\COMMAND.COM que foi encontrado em algumas instalações do Windows 2000. Se você encontrar esta mensagem de erro ao tentar iniciar o programa de instalação, consulte o site da Microsoft na Web no endereço

    http://support.microsoft.com/support/kb/articles/Q142/2/71.asp
    para obter informações sobre como resolver o problema.

  • Se você ver a seguinte mensagem de erro
    • corrupt cabinet file

    o arquivo que você baixou está corrompido. (O "cabinet file" contém arquivos de aplicativo comprimidos, dados, recursos e DLL). Verifique seu tamanho de arquivo, comparando-o com o tamanho de arquivo esperado, listado nestas instruções. Se eles não forem iguais, tente baixar o pacote novamente.

  • Se você ver a seguinte mensagem de erro
    • net.socketException: errno = 10047

    -ou-

    Unsupported version of Windows Socket API

    verifique quais são os drivers de TCP/IP que você instalou. O AppletViewer suporta somente os drivers de TCP/IP da Microsoft incluídos com o Windows 95. Se estiver usando drivers de terceiros (p.ex., Trumpet Winsock), você irá precisar mudar de volta para os drivers de TCP/IP nativos da Microsoft se desejar carregar os applets através da rede.

  • Se você ver a seguinte mensagem de erro
    • System Error during Decompression

    você pode não ter espaço suficiente no disco que contém seu diretório TEMP.

  • Se você ver a seguinte mensagem de erro
    • This program cannot be run in DOS mode.

    faça o seguinte:

    1. Abra o shell do MS-DOS (Windows/Iniciar/Programas/Prompt do MS-DOS)
    2. Clique com o botão direito na barra de título
    3. Selecione Propriedades
    4. Escolha a aba Programa
    5. Pressione o botão Avançadas...
    6. Assegure-se de que o item "Impedir que os programas de MS-DOS detectem o Windows" não esteja marcado
    7. Selecione OK
    8. Selecione OK novamente
    9. Saia do shell do MS-DOS
    10. Reinicie o computador


  • Se o AppletViewer não carregar os applets

    • você pode tentar o seguinte:
    1. set HOMEDRIVE=c:
      set HOMEPATH=\

      e reiniciar o AppletViewer (na mesma janela de linha de comando)



    2. set HOME=c:\

      3. e reiniciar o AppletViewer (na mesma janela de linha de comando)


    Se nada disso funcionar, tente:
    java -verbose sun.applet.AppletViewer

    Isto lista as classes que estão sendo carregadas. A partir desta saída, você pode determinar qual classe o AppletViewer está tentando carregar e de onde ele está tentando carregá-la. Verifique para se assegurar de que a classe existe e não está corrompida de alguma maneira.

  • Appletviewer bloqueia

    • Isto acontece com o NT Workstation 4.0, update 3, quando o DISPLAY está configurado para "true color". O appletviewer (e talvez outras entidades) trava ao começar a executar e depois ao "congelar" o sistema, consumindo 100% do processador.
    Para "testar" isto, execute o "java -verbose sun.applet.AppletView" e observe que ele trava ao tentar executar a MToolkit.class.

  • Problemas com Winsock

    • O Java 2 SDK não inclui o Microsoft Winsock 2.0. É extremamente provável que o seu sistema já tenha o Winsock 2.0. O Windows NT 4.0, Windows, ME, Windows 2000 e Windows 98 vêm com Winsock 2.0. O Windows 95 vem com o Winsock 1.1 ou 1.2, mas a maioria dos sistemas Windows 95 foram atualizados para Winsock 2.0 a esta altura.

    Para verificar qual versão do Winsock você tem, procure "winsock.dll". Depois, escolha "Propriedades" no menu Arquivo e clique na aba Versão.

    Você pode baixar o Winsock 2.0 deste endereço:

    http://www.microsoft.com/windows95/downloads/contents/wuadmintools/s_wunetworkingtools/w95sockets2/

    O URL seguinte contém informações sobre como determinar se os componentes Winsock 2.0 estão instalados em uma plataforma Windows 95:

    http://support.microsoft.com/support/kb/articles/Q177/7/19.asp

  • Criando arquivos-fonte com o Notepad

    No Microsoft Windows, quando você cria um novo arquivo no Microsoft Notepad e depois o salva pela primeira vez, o Notepad normalmente adiciona a extensão .txt ao nome do arquivo. Portanto, o arquivo que você nomeia como Test.java é salvo como Test.java.txt. É importante observar que você não pode ver a extensão .txt a menos que você ligue a visualização de extensões de arquivos (no Windows Explorer, desmarque "Ocultar extensões para os tipos de arquivos desconhecidos" em Opções de Pasta). Para evitar a extensão .txt, coloque o nome do arquivo entre aspas, como em "Test.java", ao digitá-lo na caixa de diálogo Salvar como.

    • Por outro lado, o Microsoft WordPad não adiciona extensão de arquivo se você não fornecer uma -- você deve salvar o arquivo como "Documento de texto".
