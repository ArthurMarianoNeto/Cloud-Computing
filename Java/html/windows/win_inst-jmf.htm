Para instalar o JMF Performance Pack for Windows

Execute o programa clicando duas vezes no jmf-2_1-win.exe que está localizado no diretório software-win/framework/ no nível do diretório-raiz deste CD-ROM. Ou você pode clicar no link Instalar o Java™ Media Framework a partir do CD no final desta página.

Leia as informações nesta página antes de instalar o software.

NOTA:O programa de instalação do Windows detecta automaticamente se o Netscape Communicator 4.03 ou mais recente está instalado ou não e instala o JMF para utilizá-lo!

Se estiver usando o Netscape Communicator 4.04 ou 4.05, você deve instalar o JDKTM 1.1 patch for Netscape Communicator antes de poder executar quaisquer applets de JMF.

O Netscape Communicator 4.06 ou mais recente já suporta o JDK 1.1 sem o "patch".

Instalando o JMF

  1. Verifique os requisitos de hardware e software.

  2. Desinstale o JMF 2.1.1 Beta Release se ele estiver instalado em seu sistema.

  3. Execute o programa de instalação para extrair o JMF para um diretório, como C:\Arquivos de Programas\JMF2.1.1.

  4. Verifique se seu CLASSPATH e PATH foram configurados adequadamente. Durante a instalação, o InstallShield deve configurar automaticamente seu CLASSPATH e PATH.
    set CLASSPATH=%WINDIR%\java\classes\jmf.jar;%WINDIR%\java\classes\sound.jar;.;%CLASSPATH%
É necessário configurar o seu PATH para incluir os arquivos de bibliotecas do JMF:
set PATH=%WINDIR%\System32;%PATH%    (no Windows NT)
set PATH=%WINDIR%\System;%PATH%   (no Windows 95/98)
Você pode usar o applet JMF Diagnostics para verificar se o JMF está configurado adequadamente em seu sistema.

Configurando o JMF

O JMF consiste em uma coleção de tratadores de mídia, plugins e dispositivos de captura. São configuráveis e extensíveis. Para configurar o JMF, use o aplicativo JMFRegistry. Consulte o JMF Registry User's Guide para obter mais detalhes.

Executando o JMF

Você pode executar os exemplos com o JDK Applet Viewer, com navegadores (HotJava, Netscape Communicator ou Internet Explorer) e com os aplicativos independentes JMStudio.

Executando os applets e aplicativos RTP

Você pode reproduzir somente streams unicast e broadcast com o RTPPlayerApplet. Para reproduzir um stream multicast ou para transmitir, você precisa usar o aplicativo JMStudio. Consulte a documentação desta versão do RTP para obter mais informações.

Executando o JMF com o Netscape Communicator

Pode-se usar o JMF com o Netscape Communicator 4.06 ou mais recente.

Se o Netscape Communicator já estiver instalado em seu sistema ao instalar o JMF, o programa de instalação do JMF (com sua permissão) configura o JMF para funcionar com o Netscape Communicator. Se o programa de instalação não conseguir detectar o Netscape Communicator durante a instalação, existem duas maneiras de fazer o JMF trabalhar com o Communicator:

você pode desinstalar o JMF 2.1.1 (a partir da opção Adicionar ou Remover Programas no Painel de Controle do Windows) e depois instalar o Netscape Communicator 4.06+ e o JMF 2.1.1, nesta ordem (esta é a opção mais fácil).

Instale o Netscape Communicator e depois copie as bibliotecas e as classes compartilhadas do JMF para o diretório do Netscape.

A partir do diretório %WINDIR%\Java\Classes:

Copie jmf.properties, soundbank.gm, jmf.jar e sound.jar para
C:\Arquivos de Programas\Netscape\Communicator\Program\Java\Classes

A partir do diretório %WINDIR%\System:

Copie jm*.dll e jsound.dll para
C:\Arquivos de Programas\Netscape\Communicator\Program\Java\Bin

Nota: estas instruções partem do princípio de que o Netscape Communicator estava instalado em

C:\Arquivos de Programas\Netscape.

Se não for assim, substitua os diretórios apropriados.

Se um tipo de arquivo em particular não funcionar com Netscape, primeiro verifique se o tipo MIME para este tipo de arquivo está configurado corretamente. Por default, o Netscape não fornece as configurações para os arquivos MIDI (audio/midi), RMF (audio/rmf) e GSM (audio/x-gsm). No Communicator, você pode adicionar e modificar os tipos MIME a partir do menu Edit->Preferences, na categoria Navigator/Applications.

Nota: o JMF exige acesso a diversos recursos fora da "caixa de areia" normal de Java. Quando o Communicator exibir diálogos lhe pedindo para conceder previlégios para os applets de JMF, você deve concedê-los. Para evitar a exibição repetida de diálogos de segurança, você pode habilitar o recurso "Memorize esta decisão" no diálogo de segurança.

Executando o JMF com o Internet Explorer e o Microsoft VM

O JMF funcionará na mais recente Microsoft VM (5.0.0.3177 ou posterior) com suporte a JNI. Recomenda-se o Internet Explorer 4.01 ou mais recente. O JMF também funciona no Internet Explorer 3.02 caso se utilize o Java Plug-in.

Executando o JMF com HotJava 1.1

O HotJava 1.1 parece ignorar a variável CLASSPATH. Se desejar usá-lo com o JMF, você precisará dizer explicitamente ao HotJava para usar a sua CLASSPATH. Você pode fazer isto invocando:

hotjava -classpath %CLASSPATH%

Também dê uma olhada no runme.bat no diretório em que você instalou o JMF.

Executando o JMF com o JDK 1.2 Applet Viewer

Com as permissões default do JDK 1.2, executar o JMF no appletviewer gera algumas exceções de segurança durante a execução. Você pode eliminar este problema com um arquivo de regras para conceder permissões ao JRE.  Um exemplo de arquivo de regras está incluído, jmf-policy nesta versão. Para usar o arquivo de regras com o appletviewer, invoque:

appletviewer -J-Djava.security.policy=file:<policy file> <URL>

Para obter mais informações, consulte o endereço: http://java.sun.com/products/jdk/1.2/docs/guide/security/PolicyFiles.html
